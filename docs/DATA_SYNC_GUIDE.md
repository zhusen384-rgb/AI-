# 数据同步操作指南

## 概述

本指南介绍如何在预览环境和生产环境之间同步数据。

## 数据同步工具

访问数据同步页面：
- **预览环境**: `http://localhost:5000/sync`
- **生产环境**: `https://w58x2qj4ky.coze.site/sync`

## 数据同步步骤

### 步骤 1: 在预览环境导出数据

1. 访问预览环境的数据同步页面：
   ```
   http://localhost:5000/sync
   ```

2. 点击 **"导出所有数据"** 按钮

3. 等待导出完成，系统会自动下载一个 JSON 文件
   - 文件名格式：`backup-YYYY-MM-DD.json`
   - 文件包含所有表的数据

4. 确认导出结果：
   - 检查导出的总记录数
   - 查看各表的记录数是否符合预期

### 步骤 2: 在生产环境导入数据

1. 访问生产环境的数据同步页面：
   ```
   https://w58x2qj4ky.coze.site/sync
   ```

2. 切换到 **"导入数据"** 标签

3. 选择导入方式：
   - **方式 1**: 点击 **"上传JSON文件"** 选择导出的文件
   - **方式 2**: 将 JSON 文件内容复制粘贴到文本框

4. 点击 **"导入数据"** 按钮

5. 等待导入完成，查看导入结果：
   - 总记录数
   - 成功导入数
   - 跳过数（已存在的记录）
   - 失败数（如果有的话）

### 步骤 3: 验证数据同步

1. 使用 admin 账号登录生产环境

2. 检查以下数据是否正确导入：
   - ✅ 用户列表（/users）
   - ✅ 候选人列表（/candidates）
   - ✅ 面试配置记录
   - ✅ 面试结果记录
   - ✅ 其他相关数据

## 导出的数据表

数据同步工具会导出以下表的数据：

1. **tenants** - 租户（公司/组织）
2. **users** - 用户
3. **positions** - 岗位数据
4. **loginLogs** - 登录日志
5. **invitationCodes** - 邀请码
6. **fullAiInterviewConfigs** - 全AI面试配置
7. **interviewSessions** - 面试会话
8. **fullAiInterviewResults** - 面试结果
9. **resumeEvaluationRecords** - 简历评估记录
10. **modelOptimizationHistory** - 模型优化历史
11. **candidates** - 候选人数据（含 `resumeParsedData` 简历解析结果）
12. **resumes** - 简历索引/搜索数据

## 导入选项

导入数据时，系统会自动应用以下策略：

- **skipExisting**: 跳过已存在的记录（避免重复）
- **overwrite**: 不覆盖已存在的记录
- **dryRun**: 不使用模拟运行模式（实际执行导入）

## 注意事项

### ⚠️ 安全提醒

1. **数据保密**：导出的 JSON 文件包含所有用户数据和敏感信息，请妥善保管
2. **及时删除**：导入完成后，请及时删除导出的文件
3. **访问控制**：数据同步页面需要管理员权限访问

### 📋 操作建议

1. **先备份**：导入前，建议先备份生产环境的当前数据
2. **分批导入**：如果数据量很大，可以考虑分批导入
3. **验证数据**：导入后务必验证数据的正确性
4. **测试环境**：先在测试环境测试导入流程
5. **文件存储**：如果简历原文件当前保存在本地磁盘（如 `public/uploads/resumes`），数据库同步只会迁移元数据和解析结果，不会自动打包二进制文件；建议优先配置对象存储，或额外复制 `public/uploads` 目录

### 🔧 故障排除

#### 问题 1: 导出超时

**原因**: 数据量过大，导致导出超时

**解决**:
- 分批次导出（需要修改代码）
- 检查网络连接
- 联系技术支持

#### 问题 2: 导入失败

**原因**: JSON 格式错误或数据冲突

**解决**:
- 检查 JSON 文件格式是否正确
- 确认导出文件的完整性
- 查看导入结果中的错误信息
- 尝试重新导入

#### 问题 3: 部分记录导入失败

**原因**: 数据冲突或约束违反

**解决**:
- 查看导入结果中的失败原因
- 检查失败记录的数据
- 修改导出文件后重新导入

#### 问题 4: 数据同步后无法登录

**原因**: admin 账户数据不一致

**解决**:
1. 访问：`https://w58x2qj4ky.coze.site/api/reset-admin`
2. 这会重置 admin 账户密码为 `admin123`
3. 使用 `admin` / `admin123` 登录
4. 登录后检查用户数据是否正确

## API 端点

### 导出数据

```bash
curl -X GET https://w58x2qj4ky.coze.site/api/sync/export
```

返回示例：
```json
{
  "success": true,
  "data": {
    "exportTime": "2026-03-27T01:00:00.000Z",
    "version": "1.0",
    "tables": {
      "tenants": [...],
      "users": [...],
      ...
    }
  },
  "summary": {
    "exportTime": "2026-03-27T01:00:00.000Z",
    "totalTables": 9,
    "totalRecords": 100,
    "tables": [...]
  }
}
```

### 导入数据

```bash
curl -X POST https://w58x2qj4ky.coze.site/api/sync/import \
  -H "Content-Type: application/json" \
  -d '{
    "data": {...},
    "options": {
      "skipExisting": true,
      "overwrite": false
    }
  }'
```

返回示例：
```json
{
  "success": true,
  "results": {
    "success": true,
    "totalRecords": 100,
    "imported": 95,
    "skipped": 5,
    "failed": 0,
    "tables": {...}
  }
}
```

## 常见问题

### Q: 数据同步会覆盖生产环境的数据吗？

A: 不会。导入策略是 `skipExisting`，会跳过已存在的记录，不会覆盖。

### Q: 可以只同步部分表的数据吗？

A: 当前版本不支持。需要导出所有表的数据，但导入时会自动跳过已存在的记录。

### Q: 同步后需要重新部署吗？

A: 不需要。数据同步是数据库层面的操作，不影响代码部署。

### Q: 数据同步支持增量同步吗？

A: 当前版本不支持。每次都是全量导出和导入，但导入时会跳过已存在的记录。

### Q: 数据文件很大怎么办？

A: 如果数据文件超过 100MB，建议：
1. 使用专业的数据迁移工具
2. 联系技术支持
3. 考虑分批迁移
