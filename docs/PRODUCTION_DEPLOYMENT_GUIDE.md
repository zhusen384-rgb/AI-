# 生产环境部署指南 - 管理员账户初始化

## 问题描述

如果在生产环境（https://w58x2qj4ky.coze.site）中无法使用 `admin` / `admin123` 登录，这是因为生产环境的数据库还没有初始化管理员账户。

## 解决方案

### 方法 1：使用初始化页面（推荐）

访问初始化页面：
```
https://w58x2qj4ky.coze.site/initialize
```

1. 如果是首次部署，点击 **"初始化数据库"** 按钮
2. 如果忘记密码或无法登录，点击 **"重置管理员密码"** 按钮
3. 系统会自动创建/重置管理员账户：
   - 用户名：`admin`
   - 密码：`admin123`
4. 等待 3 秒后，页面会自动跳转到登录页
5. 使用 `admin` / `admin123` 登录

### 方法 2：使用 API 端点

#### 初始化数据库（首次部署）

```bash
curl -X POST https://w58x2qj4ky.coze.site/api/init
```

#### 重置管理员密码

```bash
curl -X POST https://w58x2qj4ky.coze.site/api/reset-admin
```

## 登录信息

```
用户名：admin
密码：admin123
角色：admin
```

## 注意事项

1. **首次部署**：必须先初始化数据库才能登录
2. **密码安全**：初始化后请立即登录并修改默认密码
3. **生产环境**：确保在生产环境中执行初始化，而不是在预览环境中
4. **重复初始化**：可以重复执行初始化，不会造成数据丢失

## 验证登录

初始化后，访问：
```
https://w58x2qj4ky.coze.site/login
```

使用以下凭据登录：
- 用户名：`admin`
- 密码：`admin123`

## 故障排除

### 问题1：访问 /initialize 页面显示 404

**原因**：可能还没有部署最新代码

**解决**：等待部署完成，或者重新触发部署

### 问题2：初始化失败

**原因**：数据库连接问题

**解决**：检查环境变量配置，特别是数据库相关的配置

### 问题3：初始化成功但仍然无法登录

**原因**：可能使用了错误的凭据

**解决**：确保使用的是 `admin` / `admin123`

## API 端点说明

### `/api/init`
- **方法**：POST
- **说明**：初始化数据库，创建默认租户和管理员账户
- **权限**：公开（不需要认证）

### `/api/reset-admin`
- **方法**：POST
- **说明**：重置 admin 用户密码为 admin123
- **权限**：公开（不需要认证）

## 安全建议

1. 初始化后立即修改默认密码
2. 定期更换管理员密码
3. 启用双因素认证（如果可用）
4. 限制管理员账户的访问权限
5. 监控登录日志，发现异常及时处理
